# 🔑 API 키 관리 시스템

**간단하고 빠르게** API 키를 관리하고, 사용하지 않는 키를 정리하세요!

## ✨ 주요 기능

1. **API 키 등록** - 여러 개의 API 키를 한 곳에서 관리
2. **유효성 검증** - 클릭 한 번으로 키가 작동하는지 확인
3. **사용 위치 스캔** - 프로젝트에서 실제로 어디에 쓰이는지 자동 추적
4. **미사용 키 정리** - 안 쓰는 키는 바로 삭제

**전부 브라우저에서 가능합니다!** ⚡

## 🚀 사용 방법

**브라우저에서 `api-key-manager.html` 파일을 열기만 하면 됩니다!**

> Chrome 또는 Edge 브라우저 권장 (폴더 스캔 기능 지원)

### 1단계: API 키 등록
폼에 키 이름, API 키 값, 타입 입력 → "➕ API 키 추가" 클릭

### 2단계: 유효성 검증
"🔍 모든 키 유효성 검증 실행" 버튼 클릭 → 어떤 키가 작동하는지 확인

### 3단계: 사용 위치 스캔
"🔍 프로젝트 폴더 스캔" 버튼 클릭 → 프로젝트 폴더 선택 → 완료!

### 4단계: 정리
노란색으로 표시된 미사용 키를 "🗑️ 삭제" 버튼으로 제거

---

**끝!** 이제 프로젝트가 깔끔해졌습니다 ✨

## 📂 주요 파일

- `api-key-manager.html` - API 키 관리 시스템 (메인)
- `index.html` - Gemini API 테스트 페이지
- `scan-api-keys.js` - 터미널 스캔 스크립트 (고급 사용자용)

## 🎨 사용 예시

### 시나리오: 프로젝트 정리

1. **API 키 등록** → 10개의 키 등록
2. **유효성 검증** → "모든 키 유효성 검증" 버튼 클릭
3. **사용 위치 스캔** → "프로젝트 폴더 스캔" 버튼 클릭
4. **결과 확인 및 정리**
   - ✅ 유효 + 사용 중: 5개 → **유지**
   - ❌ 무효: 2개 → **삭제**
   - ⚠️ 미사용: 3개 → **검토 후 삭제**
5. **완료!** 깔끔하게 정리된 5개의 키만 유지

## 🛡️ 보안 주의사항

1. **API 키 노출 방지**
   - `registered-keys.json`과 `api-keys-usage.json`은 `.gitignore`에 포함됨
   - 절대 Git에 커밋하지 마세요!

2. **브라우저 저장소**
   - API 키는 브라우저의 LocalStorage에 저장됨
   - 공용 컴퓨터에서는 사용 후 데이터를 삭제하세요

3. **키 관리 권장사항**
   - 주기적으로 키 유효성 검증
   - 미사용 키는 즉시 삭제
   - 프로덕션 환경에서는 환경 변수 사용 권장

## 🔧 문제 해결

### 폴더 스캔이 작동하지 않음
- **Chrome 또는 Edge 브라우저를 사용하세요** (Safari, Firefox는 지원 안 함)
- 또는 터미널에서 `npm run scan` 사용

### API 키 검증 실패
- API 사용량 한도 확인
- 네트워크 연결 확인
